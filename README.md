# BB Visual Novel Engine — Extended

Расширение для SillyTavern, которое превращает обычный чат в более управляемую визуальную новеллу: с отношениями, памятью персонажей, вариантами действий и HUD-уведомлениями.

## Возможности

- **Social Link HUD** — боковая панель отношений с карточками персонажей.
- **Две шкалы отношений** — доверие и романтика считаются отдельно.
- **Карточки персонажей** — аватары, профили, базовые значения, статус и заметки для prompt.
- **Память персонажей** — мягкие следы, незабываемые события и архив важных моментов.
- **Кристаллизация черт** — глубокие воспоминания могут превращаться в постоянные черты характера.
- **Дневник и журнал** — история изменений, сюжетные моменты и системные записи.
- **Варианты действий** — три эмоционально размеченных ответа над полем ввода.
- **HUD-уведомления** — отдельные toast-стили для памяти, сближения, романтики, разлада и редких событий.

## Чем Extended отличается от версии maxkara14?

### БЕЗ изменений

Генерация вариантов действий, панель над чатом, toast-уведомления, утилиты, система персон, макрос `{{bb_vn}}`, кристаллизация черт, слияние дубликатов — всё работает как раньше.

7 из 11 модулей не изменились вообще: `index.js`, `state.js`, `generator.js`, `actions.js`, `toasts.js`, `utils.js`, `vn-ui.js`.

### Новое

- **Inline-редактирование памяти и черт** — записи памяти и черты характера теперь можно редактировать и удалять без необходимости лезть в консоль разработчика. Каждая запись привязана к конкретному сообщению в чате, поэтому изменения точечные.

- **Редактирование в Dev Console** — новый блок в консоли разработчика: укажите индекс сообщения, и вы увидите все связанные с сообщением события с возможностью изменить описание, эмоцию и силу влияния, или удалить событие целиком. Статы пересчитываются автоматически.

- *(WIP)* **Ручное переопределение промпта в "Система"** — можно задать свой текст социального промпта вместо автоматически генерируемого. Полезно для ручной настройки контекста отношений.

- **Два новых визуальных параметра** — сглаживание текста в дневнике (по умолчанию включено) и принудительный GPU-рендеринг.

- **Плавная автопрокрутка в HUD** — при раскрытии карточки, сворачивании или открытии "редактора" - HUD автоматически прокручивается к нужному элементу.

- **Прокрутка длинных имён** — если имя персонажа не помещается в карточку в "Связи", оно автоматически прокручивается (marquee).

### Изменения

- **Улучшенный экспорт/импорт snapshot** — при экспорте теперь сохраняются не только рассчитанные статы, но и сырые события из каждого сообщения. При импорте в другой чат всё восстанавливается с полным пересчётом от начала. Уведомление после импорта сообщает, были ли восстановлены сырые данные.
  - ⚠️ Snapshot-ы из оригинальной версии (schema v1) импортируются без проблем — используется прежний режим с baseline. Обратная совместимость v1 ↔ v2.

- **Умное архивирование незабываемых событий** — вместо безусловного переноса в архив, v2 проверяет, достаточно ли их для заполнения квоты черт. Прогресс больше не теряется из-за преждевременного архивирования.

- **Переработанный редактор персонажа** — редизайн кнопок, авторасширение текстового поля, кнопка "свернуть" и "закрыть" (закрыть редактор и журнал персонажа).

- **Обновлённый вид карточек** — убран hover-эффект с карточек, кнопка настроек персонажа переработана и перемещена под аватар, улучшен layout.

- **Улучшенный рендеринг памяти** — черты используют единые стили вместо inline-стилей, убрана обрезка длинных глубоких записей.

- **Уточнённое отображение романтики** — прогресс-бар романтики корректнее определяет, когда показывать шкалу, а когда скрывать.

## Архитектура и принцип работы

Расширение состоит из следующих модулей:

### Структура файлов

| Файл | Назначение |
|------|-----------|
| `index.js` | Точка входа. Регистрирует обработчики событий SillyTavern, инициализирует настройки, запускает автогенерацию и мониторинг смены персоны. |
| `modules/constants.js` | Константы: имя модуля, настройки по умолчанию, промпты (`SOCIAL_PROMPT`, `OPTIONS_PROMPT`), параметры toast-уведомлений. |
| `modules/state.js` | Глобальное мутабельное состояние: рассчитанные статы, токены генерации, abort-контроллер. |
| `modules/social.js` | Ядро трекера отношений: парсинг HTML-блоков из ответов модели, расчёт статов, система памяти (soft/deep/archive), слияние записей, персон-скопы, экспорт/импорт snapshot. |
| `modules/generator.js` | Генерация вариантов действий (VN Options): сборка промпта, вызов API (основной или Custom API), JSON-парсинг и ремонт, дедупликация, диверсификация тонов, кристаллизация черт, генерация описаний персонажей. |
| `modules/hud.js` | Отрисовка Social Link HUD: карточки персонажей, прогресс-бары, аватары, редактор персонажа, дневник, журнал, inline-редактирование памяти и черт. |
| `modules/actions.js` | Панель VN-действий над полем ввода: рендер option-карточек, обработка кликов, реролл, умный реролл, авто-отправка. |
| `modules/settings.js` | Страница настроек в боковой панели SillyTavern: все переключатели, Custom API, шкалы влияния, консоль разработчика, snapshot-блок. |
| `modules/toasts.js` | Система toast-уведомлений: стилизованные карточки с анимацией, пулы вариантов для разных типов событий. |
| `modules/utils.js` | Утилиты: экранирование HTML, парсинг JSON, ремонт Mojibake, нормализация данных, детектирование отказов модели, классификация тонов. |
| `modules/vn-ui.js` | UI-хелперы для кнопки генерации и контейнера вариантов. |
| `style.css` | Полный CSS: toast-уведомления, HUD-панель, карточки персонажей, VN-панель действий, анимации. |

### Основной поток данных

1. **Пользователь отправляет сообщение** → SillyTavern генерирует ответ модели.
2. В промпт (через `extension_prompt` или макрос `{{bb_vn}}`) внедряется `SOCIAL_PROMPT`, инструктирующий модель добавить скрытый `<div class="bb-vn-data">` с оценкой отношений.
3. При получении ответа (`MESSAGE_RECEIVED` / `CHARACTER_MESSAGE_RENDERED`) расширение парсит HTML-блок, извлекает `<bb-social-update>` теги и сохраняет их в `chat_metadata` привязанно к сообщению и свайпу.
4. `recalculateAllStats()` пересчитывает все статы (affinity/romance) по всей истории чата с учётом базовых значений, платонического режима, персон-скопов и snapshot-базовой линии.
5. При изменении статов генерируются **story moments** (memory/bond/fracture/tier-shift/romance и т.д.), которые отображаются как toast-уведомления.
6. **Параллельно** срабатывает автогенерация VN-вариантов (если включена): `generator.js` собирает контекст, вызывает API, парсит JSON, чинит при необходимости и рендерит карточки действий.

### Система персон-скопов

Данные отношений привязаны к **активной персоне** пользователя. При смене персоны расширение:

- Определяет текущую персону по имени и аватару (`getCurrentPersonaIdentity`).
- Строит индекс принадлежности scope-ключей к персон-рефам (`buildScopeOwnershipIndex`).
- Выбирает подходящий scope (`choosePersonaScopeCandidate`) на основе очков совпадения.
- Поддерживает алиасы для корректной связи старых и новых scope-ключей.
- Автоматически очищает записи, принадлежащие персоне самого пользователя (`purgeUserPersonaTracking`).

### Snapshot-система (экспорт/импорт)

- **Экспорт** (schema_version: **2**): сохраняет рассчитанные статы, глобальный журнал, story moments, базовые значения, базу персонажей и **сырые обновления из сообщений** (`raw_message_updates`) — все `bb_social_swipes` и `bb_vn_char_traits_swipes`.
- **Импорт**: устанавливает snapshot как базовую линию, восстанавливает сырые обновления в сообщения (с маппингом scope-ключей), и продолжаёт считать только новые события поверх snapshot.
- **Обратная совместимость**: snapshot формата v1 (schema_version: 1, без `raw_message_updates`) импортируется корректно — используется legacy-режим с baseline и cutoff.

## Варианты действий

VNE может генерировать три действия для пользователя с разными намерениями и тоном. Каждый вариант содержит:

- `intent` — короткая суть действия (2–5 слов, русский);
- `tone` — эмоциональная окраска (1–2 слова, русский);
- `forecast` — краткий намёк на возможное последствие (3–9 слов);
- `targets` — персонажи, которых действие затронет сильнее всего (1–3 имени);
- `risk` — устаревшее поле для обратной совместимости;
- `message` — готовый текст ответа (roleplay-формат с *действиями* и «диалогом»).

### Управление вариантами

- **Генерация** — первая генерация вариантов через кнопку или авто-показ.
- **Реролл** — повторная генерация с исключением текущих intents/tones.
- **Умный реролл** — генерация с пользовательским пожеланием (через popup).
- **Длина ответа** — `short` (~450–900 символов), `medium` (~900–1600), `long` (~1800–3200).
- **Авто-показ** — автоматическая генерация вариантов после каждого ответа модели (с ретраями и проверкой готовности UI).

### Надёжность генерации

- JSON-парсинг с автоматическим ремонтом (удаление markdown-заборов, экранирование строк, восстановление баланса скобок).
- Повторная генерация при невалидном JSON через `repairOptionsJson`.
- Заполнение недостающих вариантов (`fillMissingOptions` — до 2 попыток).
- Диверсификация тонов при их повторении (`diversifyOptionTones`).
- Дедупликация по intent + message.

## Настройки и интеграция

### Переключатели

| Настройка | По умолчанию | Описание |
|-----------|:---:|----------|
| Авто-отправка при выборе | ✓ | Автоматически отправлять выбранный вариант |
| Авто-показ вариантов действий | ✗ | Автоматически генерировать варианты после ответа модели |
| Тон и прогноз вариантов | ✓ | Показывать emotional choice framing (tone + forecast) |
| Отключить трекер отношений | ✗ | Отключить трекер, оставив генерацию вариантов |
| Сглаживание текста в Дневнике | ✓ | Антиалиасинг рендера карточек дневника |
| Принудительный GPU-рендеринг | ✗ | GPU-ускорение CSS-анимаций в карточках (если текст скачет) |

### Custom API

Поддерживается OpenAI-compatible Custom API для генерации вариантов и служебных запросов. Если Custom API не отвечает или возвращает ошибку, автоматически происходит fallback на основную модель SillyTavern с toast-уведомлением.

Настройки:
- URL (по умолчанию: `https://api.groq.com/openai/v1`)
- API-ключ
- Выбор модели (автозагрузка списка через `/models`)

### Макрос `{{bb_vn}}`

Ручная интеграция через макрос — расширение заменяет `{{bb_vn}}` в `generate_data` (сообщения и корневой объект) на актуальный текст социальных данных. Работает только если трекер отношений включён.

### Кастомная шкала отношений

Сила влияния событий настраивается отдельно для доверия и романтики:

| Токен | Дружба (по умолч.) | Романтика (по умолч.) |
|-------|:---:|:---:|
| `unforgivable` | −20 | −20 |
| `major_negative` | −8 | −8 |
| `minor_negative` | −2 | −2 |
| `none` | 0 | 0 |
| `minor_positive` | +2 | +2 |
| `major_positive` | +8 | +8 |
| `life_changing` | +20 | +20 |

Обе шкалы можно сбросить к значениям по умолчанию одной кнопкой.

## Память и персонажи

### Типы памяти

| Тип | Порог Δ | Поведение |
|-----|:---:|----------|
| **Мягкие следы** (soft) | \|Δ\| ≥ 2 | Временные следы недавних событий, хранятся до 4 записей |
| **Незабываемые события** (deep) | \|Δ\| ≥ 15 | Постоянные следы значимых событий |
| **Архив** | — | Перенесённые старые deep-записи при достижении лимита |

### Черты характера (кристаллизация)

При накоплении 5+ глубоких воспоминаний одного тона (положительных или отрицательных) пользователь может кристаллизовать черту характера. Кристаллизация генерирует черту через LLM, валидирует и сохраняет её. Черты бывают:
- **Светлые** (positive) — из 5+ положительных deep-памятей
- **Мрачные** (negative) — из 5+ отрицательных deep-памятей

### Профиль персонажа

Для каждого персонажа можно:
- Загрузить аватар с кадрированием (фокус по X/Y, масштаб)
- Сгенерировать 10-строчный структурированный профиль по шаблону (имя, возраст, роль, внешность, характер, манера речи, прошлое, отношение к пользователю, сценический гайд)
- Задать базовые значения доверия и романтики
- Включить платонический режим (блокирует романтическую линию)
- Переименовывать, скрывать и восстанавливать персонажей

### Данные и привязка

- Данные привязаны к **сообщениям**, **свайпам** и **активной персоне**.
- При смене персоны автоматически подгружается соответствующий scope.
- При слиянии дубликатов персонажей (fuzzy matching) все записи переносятся на целевого персонажа.

## Консоль разработчика

Встроенный набор инструментов для отладки:

- Инъекция тестовых данных: дружба ±, романтика ±, глубокие следы (светлый/мрачный/смешанный), черты
- Ручная смена статуса к пользователю
- Слияние дубликатов персонажей (с авто-подсказками по fuzzy matching)
- Полный сброс персонажа или всей истории
- **Редактирование событий**: загрузка событий конкретного сообщения по индексу, изменение описания, эмоции и импакта с пересчётом статов
- Удаление отдельных событий

## HUD-уведомления

Расширение использует собственную систему toast-уведомлений с анимацией и стилизацией по типу:

| Тип | Бейдж | Стиль | Цвет |
|-----|-------|-------|------|
| deep-positive | Память | memory | Фиолетовый |
| deep-negative | Шрам | fracture | Красный |
| soft-positive | Отголосок | bond | Зелёный |
| soft-negative | Осадок | negative | Розовый |
| tier-shift | Маршрут | system | Синий |
| status-shift | Поворот | milestone | Янтарный |
| romance-positive | Искра | romance | Розовый |
| romance-negative | Надлом | fracture | Малиновый |
| crystallized (positive) | Кристалл | legendary | Золотой |
| crystallized (negative) | Излом | fracture | Розовый |
| intro | Трекер | system | Голубой |

Каждый тип имеет пул из 4–5 вариантов заголовков и текстов, выбираемых детерминированно по seed.

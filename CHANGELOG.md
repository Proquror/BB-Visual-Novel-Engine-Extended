Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [3.4.0] - 22 June 2026

### Added

- **Отключение генератора вариантов действий** — возможность полностью выключить генератор VN-вариантов (аналогично отключению трекера отношений). При отключении панель «Действия VN» скрывается целиком, точка повторного включения — кнопка 🎬 в тулбаре HUD, inline-тумблер у поля ввода или pill в настройках. Гард в `bbVnGenerateOptionsFlow` / `restoreVNOptions` / `clearVNOptions` и в авто-генерации (`index.js`) не даёт генератору работать в отключённом состоянии. ([constants.js], [actions.js], [generator.js], [settings.js], [hud.js], [index.js])
  - Новая кнопка тулбара **🎬 Генератор** — переключает `disableGenerator`, синхронизируется через `updateGeneratorToolbarButton()` (вызывается также из `updateHudVisibility()`). ([hud.js])
  - Новый inline-тумблер у поля ввода — `#bb-vn-btn-toggle-generator`; в отключённом состоянии разворачивается в подписанную кнопку «Генератор выключен», давая очевидную точку повторного включения прямо на месте основной кнопки. ([actions.js], [style.css])
  - Новый pill в настройках — «Отключить генератор действий» с change-хендлером и восстановлением сохранённых вариантов при включении. ([settings.js])
  - Новая настройка `disableGenerator` (по умолчанию `false`) в `DEFAULT_SETTINGS`. ([constants.js])

- **Плавающее позиционирование popup-оверлеев тулбара** — overlay больше не fullscreen-бэкдроп. Теперь это прозрачный `position: fixed` контейнер, который JS позиционирует под кнопкой, открывшей его (`positionToolbarPopup(btn)`), с clamping во viewport и auto-flip над кнопкой при нехватке места снизу. ([hud.js], [style.css])
  - Overlay перенесён из шаблона HUD в корневой stacking context (`<body>`) — больше не клиппится под topbar SillyTavern (z-index 3005) и не наследует `display:none` от свёрнутого inline-drawer ST. ([hud.js])
  - `z-index` overlay поднят с `895` до `100000`. ([style.css])
  - Авто-перепозиционирование при `resize` и `scroll` (capture phase), а также при изменении размеров содержимого popup через `ResizeObserver` — popup остаётся привязан к кнопке и не «уплывает» в устаревший угол при раскрытии inline-drawer внутри. ([hud.js])
  - Закрытие по клику вне popup переведено с backdrop-клика на `document`-listener (`click.bbToolbarPopup`) — overlay стал `pointer-events: none`, поэтому клик «по фону» ловится на уровне документа. ([hud.js])

### Changed

- **JSON-схема больше не отправляется в основной API** — `getVnOptionsJsonSchema()` (strict `{"options":[...]}`) прикрепляется только для Custom API. Для основного API SillyTavern `jsonSchema` убран: пост-процессор ST больше не подменяет валидный bare-array ответ модели на пустой `{}` на Anthropic-via-ST, локальных моделях (llama.cpp / Ollama), Gemini и reverse-proxy. Существующий пайплайн `parseModelJson` + `extractBalancedSegment` + `repairJsonCandidate` разбирает любой формат. ([generator.js])
- **`repairOptionsJson` переключён на `responseFormat: 'text'`** — раньше `responseFormat: 'json'` + `jsonSchema` противоречили друг другу (промпт просил bare array, схема описывала object). Теперь модель возвращает bare array без вмешательства, а `parseModelJson` достаёт его через `extractBalancedSegment`. ([generator.js])
- **CSS popup-оверлея** — убран fullscreen-бэкдроп (`rgba(0,0,0,0.5)` + `backdrop-filter: blur(4px)` + `padding: 40px 14px`), overlay стал прозрачным `pointer-events: none` контейнером; popup получил flex-layout с `width: min(420px, calc(100vw - 16px))` и `max-height: min(70vh, 560px)`, тело — `flex: 1 1 auto; min-height: 0; overflow-y: auto` вместо жёсткого `max-height: 60vh`. ([style.css])
- **Принудительный `display: block` для settings shell** при открытии popup — ST прячет `inline-drawer-content` через `display:none` в свёрнутом состоянии; без форсирования перенесённые в popup элементы наследовали нулевые размеры, и popup оказывался невидимым на мобильных. ([hud.js])
- **OpenAI native (gpt-4o / gpt-4.1 / o1)** — раньше работал через strict schema, теперь работает через `parseModelJson`; теряет strict-schema гарантию, но на практике `OPTIONS_PROMPT` + `parseModelJson` + `fillMissingOptions` + `dedupeOptions` + `diversifyOptionTones` уже покрывали все формы ответа. Видимых пользователю изменений нет. Custom API (Groq / OpenRouter / прямые OpenAI-прокси) — без изменений, схема и strict mode сохранены. ([generator.js])
- **Версия** — `3.3.0` → `3.4.0`. ([manifest.json])

### Fixed

- **Пустой ответ основного API не уходит в ремонт** — новый гард `isEmptyRepairInput()` ловит `{}`, `[]` и markdown-fences вокруг них; ремонт пропускается, выбрасывается читаемая ошибка («Модель вернула пустой ответ…») вместо вызова модели на пустом `BROKEN INPUT:` и галлюцинации вариантов из истории чата. Тот же гард стоит на основном `result` перед парсингом. ([generator.js])
- **Генерация вариантов на Anthropic-via-ST / локальных моделях / Gemini / reverse-proxy** — возвращала пустоту или фантомные варианты (`intent = "Новый ход"` / «Модель вернула поврежденный JSON»); исправлено удалением strict json_schema из основного API. Воспроизводилось идентично на немодифицированном оригинале, что подтверждает архитектурную природу бага. ([generator.js])
- **Popup тулбара для мобильных юзеров** — центрированный модал заменён на floating-panel паттерн с привязкой к кнопке. ([hud.js], [style.css])
- **Логирование полного сырого ответа** при ошибке парсинга JSON — раньше `logOptionsJsonFailure` сохранял ~260 символов вокруг позиции ошибки, чего не хватало для диагностики реальных сбоев на Anthropic / OpenAI прокси. Теперь в `console.warn` пишется `fullResult` целиком. ([generator.js])

## [3.3.0] - 16 June 2026

### Added

- **Тулбар HUD** — 5 кнопок в заголовке HUD (⚙️ Настройки, 🛠 Dev, 📤 Экспорт, 📥 Импорт, 👁️‍🗨️ Скрытые персонажи), открывающих popup-оверлеи вместо боковой панели SillyTavern. ([hud.js])
- **Отключение/включение трекера из HUD** — клик по индикатору статуса («активно» / «отключено») или по иконке в оверлее переключает состояние трекера. При отключении показывается оверлей с кнопкой включения. ([hud.js])
- **Экспорт `getAutoGeneratedDataText()`** — публичная функция для получения текста авто-генерируемых данных промпта (сегменты 2–5). Используется в HUD для read-only превью. ([social.js])
- **Предпросмотр промпта в Журнале** — раскрывающийся блок «Inject Prompt» показывает инструкции и данные персонажей раздельно, с кнопками копирования и редактирования. ([hud.js])
- **Бейджи дельт памяти** — маленькие индикаторы `+N`/`−N` на таблетках памяти, показывающие изменение доверия и романтики для каждого события. ([hud.js])
- **Предзапросный пересчёт статов** — новые обработчики событий `GENERATION_AFTER_COMMANDS` и `GENERATE_BEFORE_COMBINE_PROMPTS` вызывают `recalculateAllStats()` перед сборкой промпта, гарантируя актуальные данные. ([index.js])

### Changed

- **Редактор инструкций промпта** — раздельный просмотр и редактирование: инструкции (сегмент 1 — можно переопределить через `bb_vn_prompt_instructions_override`) + данные персонажей (сегменты 2–5, авто-генерация, только чтение). Кнопки «Сохранить» и «Сбросить к дефолту». ([hud.js], [social.js])
- **Исправление изоляции свайпов** — переработка `scanAndCleanMessage()` и `recalculateAllStats()`: определение правильного слота свайпа через сопоставление исходного текста тега VNE с `msg.swipes[]`, корректировка устаревшего `msg.swipe_id`, защита от stale-данных при свайпе в процессе, fresh-VNE fallback. ([social.js])
- **Экспорт/импорт Snapshot v2** — `exportActivePersonaSnapshot()` теперь сохраняет `raw_message_updates` (все `bb_social_swipes` и `bb_vn_char_traits_swipes` по индексам сообщений). `importActivePersonaSnapshot()` восстанавливает сырые обновления с маппингом scope-ключей и полным пересчётом. Обратно совместим с v1. ([social.js])
- **Умное архивирование незабываемых событий** — глубокие воспоминания архивируются только если их достаточно для заполнения квоты черт, предотвращая преждевременную потерю видимого прогресса. ([social.js])
- **Обогащённые метаданные** — записи памяти, moments и записи глобального журнала теперь содержат `msgIndex`, `updateIdx`, `affinityDelta`, `romanceDelta`, `charName`, `reason` и сырые импакт-значения для полной трассировки. ([social.js], [hud.js])
- **Редизайн редактора персонажа** — кнопки сохранения и скрытия в виде градиентных пиллей, кнопки «свернуть» и «закрыть», авторасширение текстового поля описания, улучшенный layout. ([hud.js], [style.css])
- **Стили popup-секций тулбара** — настройки, Dev Console, экспорт/импорт и скрытые персонажи перенесены из боковой панели в popup-оверлеи с соответствующими scoped-стилями. ([hud.js], [style.css])
- **Версия** — `3.2.0` → `3.3.0`. ([manifest.json])

### Removed

- **Полный оверрайд промпта (`bb_vn_prompt_override`)** — устаревший ключ, заменявший весь промпт целиком, удалён из логики и UI. Вместо него используется `bb_vn_prompt_instructions_override` (только инструкции, данные персонажей всегда авто-генерируются). ([social.js], [hud.js], [style.css])
- **Legacy-миграция** — кнопка «Конвертировать» и CSS для legacy-лейбла удалены. ([hud.js], [style.css])

## [3.2.0] - 9 June 2026

### Added

- **Inline-редактирование памяти** — мягкие и глубокие воспоминания теперь можно редактировать и удалять прямо в карточке персонажа через inline-формы. Каждая запись привязана к индексу сообщения, изменения точечные. ([hud.js], [social.js])
- **Inline-редактирование черт** — кристаллизованные черты характера можно редактировать и удалять без консоли разработчика. ([hud.js], [social.js])
- **Inline-редактирование дневниковых записей** — story moments в дневнике событий можно редактировать через встроенную форму. ([hud.js])
- **Inline-редактирование и удаление записей журнала** — глобальный журнал (Журнал → системные записи) поддерживает редактирование и удаление с пересчётом статов. ([hud.js], [social.js])
- **Редактор событий в Dev Console** — новый блок: укажите индекс сообщения, загрузите все связанные события, измените описание / эмоцию / импакт или удалите событие. Статы пересчитываются автоматически. ([settings.js], [social.js])
- **Редактор инструкций промпта** — просмотр и редактирование через `bb_vn_prompt_override`. Кнопки «Сохранить» и «Сбросить к дефолту». ([hud.js], [social.js]? не помню, пишу задним числом)
- **Предпросмотр промпта в Журнале** — раскрывающийся блок «Inject Prompt» показывает инструкции и данные персонажей раздельно, с кнопками копирования и редактирования. ([hud.js])
- **Бейджи дельт памяти** — маленькие индикаторы `+N`/`−N` на таблетках памяти, показывающие изменение доверия и романтики для каждого события. ([hud.js])
- **Пагинация памяти** — глубокие воспоминания показывают до 4 записей, остальные раскрываются по кнопке «Показать ещё». ([hud.js])
- **Марки прокрутки имён** — длинные имена персонажей автоматически прокручиваются (marquee), если не помещаются в карточку. ([hud.js])
- **Плавный автофокус** — при раскрытии/свёртывании карточки или редактора HUD автоматически прокручивается к элементу. ([hud.js])
- **Сглаживание текста в Дневнике** — новая настройка (по умолчанию вкл.), применяет `-webkit-font-smoothing: antialiased` к карточкам дневника. ([settings.js], [style.css])
- **Принудительный GPU-рендеринг** — новая настройка (по умолчанию выкл.), принудительно включает GPU-ускорение CSS для карточек дневника. ([settings.js], [style.css])

### Changed

- **Экспорт/импорт Snapshot v2** — `exportActivePersonaSnapshot()` теперь сохраняет `raw_message_updates` (все `bb_social_swipes` и `bb_vn_char_traits_swipes` по индексам сообщений). `importActivePersonaSnapshot()` восстанавливает сырые обновления с маппингом scope-ключей и полным пересчётом. - **Умное архивирование незабываемых событий** — глубокие воспоминания архивируются только если их достаточно для заполнения квоты черт, предотвращая преждевременную потерю видимого прогресса. ([social.js])

- **Улучшенный рендеринг памяти** — черты используют единые CSS-классы вместо inline-стилей, убрана обрезка длинных глубоких записей. ([hud.js], [style.css])
- **Уточнённое отображение романтики** — прогресс-бар романтики корректнее определяет, когда показывать шкалу. ([hud.js], [style.css])

## [3.1.0] - 5 June 2026

### Added

- Исходный релиз BB Visual Novel Engine 3.1.0 от BruniikBron.
- Social Link HUD с карточками персонажей.
- Две шкалы отношений: доверие и романтика.
- Система памяти: мягкие следы, незабываемые события, архив.
- Кристаллизация черт характера.
- Варианты действий (VN Options) с генерацией, рероллом и умным рероллом.
- HUD-уведомления (toast) с типизацией.
- Система персон-скопов.
- Экспорт/импорт snapshot (schema v1).
- Макрос `{{bb_vn}}`.
- Custom API для генерации.
- Консоль разработчика.

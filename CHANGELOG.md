# Changelog

Все заметные изменения проекта BB Visual Novel Engine Extended документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

---

## [3.3.0] - 16 June 2026

### Added

- **Тулбар HUD** — 5 кнопок в заголовке HUD (⚙️ Настройки, 🛠 Dev, 📤 Экспорт, 📥 Импорт, 👁️‍🗨️ Скрытые персонажи), открывающих popup-оверлеи вместо боковой панели SillyTavern. ([hud.js])
- **Отключение/включение трекера из HUD** — клик по индикатору статуса («активно» / «отключено») или по иконке в оверлее переключает состояние трекера. При отключении показывается оверлей с кнопкой включения. ([hud.js])
- **Экспорт `getAutoGeneratedDataText()`** — публичная функция для получения текста авто-генерируемых данных промпта (сегменты 2–5). Используется в HUD для read-only превью. ([social.js])
- **Предпросмотр промпта в Журнале** — раскрывающийся блок «Inject Prompt» показывает инструкции и данные персонажей раздельно, с кнопками копирования и редактирования. ([hud.js])
- **Бейджи дельт памяти** — маленькие индикаторы `+N`/`−N` на таблетках памяти, показывающие изменение доверия и романтики для каждого события. ([hud.js])
- **Предзапросный пересчёт статов** — новые обработчики событий `GENERATION_AFTER_COMMANDS` и `GENERATE_BEFORE_COMBINE_PROMPTS` вызывают `recalculateAllStats()` перед сборкой промпта, гарантируя актуальные данные. ([index.js])

### Changed

- **Редактор инструкций промпта** — раздельный просмотр и редактирование: инструкции (сегмент 1 — можно переопределить через `bb_vn_prompt_instructions_override`) + данные персонажей (сегменты 2–5, авто-генерация, только чтение). Кнопки «Сохранить» и «Сбросить к дефолту». ([hud.js], [social.js])
- **Исправление изоляции свайпов** — переработка `scanAndCleanMessage()` и `recalculateAllStats()`: определение правильного слота свайпа через сопоставление исходного текста тега VNE с `msg.swipes[]`, корректировка устаревшего `msg.swipe_id`, защита от stale-данных при свайпе в процессе, fresh-VNE fallback. ([social.js])
- **Экспорт/импорт Snapshot v2** — `exportActivePersonaSnapshot()` теперь сохраняет `raw_message_updates` (все `bb_social_swipes` и `bb_vn_char_traits_swipes` по индексам сообщений). `importActivePersonaSnapshot()` восстанавливает сырые обновления с маппингом scope-ключей и полным пересчётом. Обратно совместим с v1. ([social.js])
- **Умное архивирование незабываемых событий** — глубокие воспоминания архивируются только если их достаточно для заполнения квоты черт, предотвращая преждевременную потерю видимого прогресса. ([social.js])
- **Обогащённые метаданные** — записи памяти, moments и записи глобального журнала теперь содержат `msgIndex`, `updateIdx`, `affinityDelta`, `romanceDelta`, `charName`, `reason` и сырые импакт-значения для полной трассировки. ([social.js], [hud.js])
- **Редизайн редактора персонажа** — кнопки сохранения и скрытия в виде градиентных пиллей, кнопки «свернуть» и «закрыть», авторасширение текстового поля описания, улучшенный layout. ([hud.js], [style.css])
- **Стили popup-секций тулбара** — настройки, Dev Console, экспорт/импорт и скрытые персонажи перенесены из боковой панели в popup-оверлеи с соответствующими scoped-стилями. ([hud.js], [style.css])
- **Версия** — `3.2.0` → `3.3.0`. ([manifest.json])

### Removed

- **Полный оверрайд промпта (`bb_vn_prompt_override`)** — устаревший ключ, заменявший весь промпт целиком, удалён из логики и UI. Вместо него используется `bb_vn_prompt_instructions_override` (только инструкции, данные персонажей всегда авто-генерируются). ([social.js], [hud.js], [style.css])
- **Legacy-миграция** — кнопка «Конвертировать» и CSS для legacy-лейбла удалены. ([hud.js], [style.css])

## [3.2.0] - 9 June 2026

### Added

- **Inline-редактирование памяти** — мягкие и глубокие воспоминания теперь можно редактировать и удалять прямо в карточке персонажа через inline-формы. Каждая запись привязана к индексу сообщения, изменения точечные. ([hud.js], [social.js])
- **Inline-редактирование черт** — кристаллизованные черты характера можно редактировать и удалять без консоли разработчика. ([hud.js], [social.js])
- **Inline-редактирование дневниковых записей** — story moments в дневнике событий можно редактировать через встроенную форму. ([hud.js])
- **Inline-редактирование и удаление записей журнала** — глобальный журнал (Журнал → системные записи) поддерживает редактирование и удаление с пересчётом статов. ([hud.js], [social.js])
- **Редактор событий в Dev Console** — новый блок: укажите индекс сообщения, загрузите все связанные события, измените описание / эмоцию / импакт или удалите событие. Статы пересчитываются автоматически. ([settings.js], [social.js])
- **Редактор инструкций промпта** — просмотр и редактирование через `bb_vn_prompt_override`. Кнопки «Сохранить» и «Сбросить к дефолту». ([hud.js], [social.js]? не помню, пишу задним числом)
- **Предпросмотр промпта в Журнале** — раскрывающийся блок «Inject Prompt» показывает инструкции и данные персонажей раздельно, с кнопками копирования и редактирования. ([hud.js])
- **Бейджи дельт памяти** — маленькие индикаторы `+N`/`−N` на таблетках памяти, показывающие изменение доверия и романтики для каждого события. ([hud.js])
- **Пагинация памяти** — глубокие воспоминания показывают до 4 записей, остальные раскрываются по кнопке «Показать ещё». ([hud.js])
- **Марки прокрутки имён** — длинные имена персонажей автоматически прокручиваются (marquee), если не помещаются в карточку. ([hud.js])
- **Плавный автофокус** — при раскрытии/свёртывании карточки или редактора HUD автоматически прокручивается к элементу. ([hud.js])
- **Сглаживание текста в Дневнике** — новая настройка (по умолчанию вкл.), применяет `-webkit-font-smoothing: antialiased` к карточкам дневника. ([settings.js], [style.css])
- **Принудительный GPU-рендеринг** — новая настройка (по умолчанию выкл.), принудительно включает GPU-ускорение CSS для карточек дневника. ([settings.js], [style.css])

### Changed

- **Экспорт/импорт Snapshot v2** — `exportActivePersonaSnapshot()` теперь сохраняет `raw_message_updates` (все `bb_social_swipes` и `bb_vn_char_traits_swipes` по индексам сообщений). `importActivePersonaSnapshot()` восстанавливает сырые обновления с маппингом scope-ключей и полным пересчётом. Обратно совместим с v1. ([social.js])
- **Умное архивирование незабываемых событий** — глубокие воспоминания архивируются только если их достаточно для заполнения квоты черт, предотвращая преждевременную потерю видимого прогресса. ([social.js])
- **Редизайн карточек персонажей** — убран hover-эффект, кнопка настроек переработана и перемещена под аватар, улучшен layout, кнопка раскрытия стала полноширинной с анимацией поворота иконки, анимация раскрытия через `grid-template-rows: 0fr/1fr` вместо `max-height`. ([hud.js], [style.css])
- **Редизайн редактора персонажа** — кнопки сохранения и скрытия в виде градиентных пиллей, кнопки «свернуть» и «закрыть», авторасширение текстового поля описания, улучшенный layout. ([hud.js], [style.css])
- **Улучшенный рендеринг памяти** — черты используют единые CSS-классы вместо inline-стилей, убрана обрезка длинных глубоких записей. ([hud.js], [style.css])
- **Уточнённое отображение романтики** — прогресс-бар романтики корректнее определяет, когда показывать шкалу. ([hud.js], [style.css])
- **Стили popup-секций тулбара** — настройки, Dev Console, экспорт/импорт и скрытые персонажи перенесены из боковой панели в popup-оверлеи с соответствующими scoped-стилями. ([hud.js], [style.css])
- **Версия** — `3.1.0` → `3.2.0`, автор обновлён на `BruniikBron (mod by Proquror)`. ([manifest.json])
- **Inline-стили карточек** — все inline-стили из шаблонов HUD перенесены в CSS-классы. ([hud.js], [style.css])

## [3.1.0] - 5 June 2026

### Added

- Исходный релиз BB Visual Novel Engine 3.1.0 от BruniikBron.
- Social Link HUD с карточками персонажей.
- Две шкалы отношений: доверие и романтика.
- Система памяти: мягкие следы, незабываемые события, архив.
- Кристаллизация черт характера.
- Варианты действий (VN Options) с генерацией, рероллом и умным рероллом.
- HUD-уведомления (toast) с типизацией.
- Система персон-скопов.
- Экспорт/импорт snapshot (schema v1).
- Макрос `{{bb_vn}}`.
- Custom API для генерации.
- Консоль разработчика.
